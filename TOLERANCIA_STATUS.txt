✅ 2. MATCHING EM CASCATA (4 ESTRATÉGIAS)

   [1] EXATO (100%)
       └─ Match perfeito após normalização
       └─ "tipo_ativo" = "tipo_ativo"
       └─ Score: 1.00

   [2] SINÔNIMO (95%)
       └─ Encontrado em dicionário (17 campos × ~150+ aliases)
       └─ "proprietario" → "usuario_responsavel"
       └─ "data entrada" → "data_entrada"
       └─ Score: 0.95

   [3] SIMILARIDADE (60–85%)
       └─ Difflib.SequenceMatcher com penalidades
       └─ "tipo de ativo" → "tipo_ativo" (similaridade 95%, score 0.85)
       └─ Detecta ambigüidade (reduz score em 15%)
       └─ Score: 0.60 a 0.85 (dinâmico)

   [4] REJEITA (<60%)
       └─ Coluna ignorada (não bloqueia)
       └─ "xyz_coluna_aleatoria" → Ignorado
       └─ Score: 0.0

✅ 4. SCORING DE CONFIANÇA (0.0–1.0)

   1.00 → Correspondência exata (verde, auto ✓)
   0.95 → Sinônimo direto (verde, auto ✓)
   0.85 → Similaridade forte (amarelo, confirma)
   0.80 → Similaridade forte (amarelo, confirma)
   0.75 → Similaridade média (amarelo, confirma)
   0.70 → Similaridade média (laranja, sugestão)
   0.60 → Similaridade fraca (laranja, sugestão)
   <0.60 → Insuficiente (cinza, ignorado)

✅ 5. REGRAS DE BLOQUEIO E CONFIRMAÇÃO

   CAMPO CRÍTICO (tipo_ativo, marca, modelo, setor, data_entrada):
   ├─ Score ≥ 90%: AUTO ✓ (checkbox pré-marcada)
   ├─ Score 75–89%: CONFIRMA ⚠️ (checkbox pré-marcada)
   └─ Score <75%: BLOQUEIA ❌ (botão "Confirmar" desabilitado)

   CAMPO OBRIGATORIO_COM_INFERENCIA (status, categoria, descrição):
   ├─ Score ≥ 90%: AUTO ✓
   ├─ Score 75–89%: CONFIRMA ⚠️
   ├─ Score 60–74%: SUGESTÃO (checkbox vazia)
   └─ Score <60%: IGNORA

   CAMPO OPCIONAL (serial, garantia, fornecedor, etc):
   ├─ Score ≥ 75%: AUTO ✓
   ├─ Score 60–74%: SUGESTÃO (checkbox vazia)
   └─ Score <60%: IGNORA

✅ 6. PENALIDADES APLICADAS

   Ambigüidade em Similaridade:
   └─ Se 2+ campos com scores muito próximos (<10% diferença)
   └─ Reduz score em 15%
   └─ Sinaliza: "[⚠️ Ambíguo: múltiplos matches similares]"

   Colisão (Múltiplas Colunas → Mesmo Campo):
   └─ Detecta e reduz score das duplicatas em 20%
   └─ Mantém apenas a com maior score original
   └─ Sinaliza: "[Colisão: campo 'X' já mapeado com score maior]"

✅ 7. INTEGRAÇÃO COM PREVIEW

   gerar_preview_estruturado() retorna JSON com:

   {
     "colunas": {
       "exatas": [{...com scores, estratégia, motivo, ação esperada...}],
       "sugeridas": [{...}],
       "ignoradas": [{...}]
     },
     "resumo_validacao": {
       "total_colunas": N,
       "colunas_mapeadas_alta": N,
       "colunas_mapeadas_media": N,
       "colunas_mapeadas_baixa": N,
       "colunas_ignoradas": N,
       "campos_criticos_faltantes": [...],
       "bloqueada": bool,
       "requer_confirmacao": bool
     },
     "avisos": [
       "⚠️ BLOQUEIO: Campos críticos...",
       "ℹ️ Confirmação necessária: X mapeamento(s)..."
     ]
   }

   Informações por match:
   ├─ coluna_origem
   ├─ campo_destino
   ├─ score (0.0–1.0)
   ├─ score_percentual (0–100%)
   ├─ estrategia (exata, sinonimo, similaridade, nao_mapeado)
   ├─ motivo (descrição textual)
   ├─ acao_esperada (auto_aplicar, sugerir, confirma, bloqueia, ignorar)
   ├─ requer_confirmacao (bool)
   ├─ classe_checkbox (high_confidence, medium_confidence, low_confidence, blocklist, ignored)
   └─ Booleanos de confiança (confianca_alta, confianca_media, confianca_baixa)
